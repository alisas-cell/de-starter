为什么不只用一句提示词

一句提示词可以开始，
Skill 负责把安全流程走完。

两者不是互相取代：提示词表达这一次的意图，Skill 固化每一次都应该遵守的执行边界。
普通对话提示词

能启动任务，但依赖当次对话

结果取决于用户有没有想到全部要求，以及当前 Agent 有没有逐项补问。
1每个新项目都要重新解释保护项、审批方式和回滚要求
2容易漏掉许可证、功能键、空目录或状态过期等边界
3步骤可能随上下文变化，难以让别人复现和审查
4一句“继续”可能被误解为允许写入
对比
de-starter Skill

把安全边界变成可复用流程

Agent 仍然负责判断与执行，但必须按同一套协议留下证据。
✓固定 P0–P3 分级，两道人工审批门，不把催促当批准
✓预览与备份放在项目外，避免安全材料污染源码
✓错误令牌和过期预览自动拒绝，防止部分执行
✓执行后再次验证，并生成恢复清单和反向差异
提示词是起点，Skill 是可复用的执行边界。低风险不等于零风险 · 用户仍要检查并批准决定
