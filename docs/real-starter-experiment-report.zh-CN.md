## 1. 实验目的

验证 `de-starter` 是否能在不破坏真实产品能力的前提下，把购买 Starter 中的卖家身份、演示残留和 source-named 空目录处理为可解释、可审批、可恢复的结果。

本实验不把“关键词清零”当作成功。成功标准同时要求：

1. P0 法律、版权、疑似凭据证据保持不变；
2. P1 计费、认证、数据库、API 和持久化标识只在有迁移与回滚计划时修改；
3. P2 Demo、样例、评价、测试数据与资产由用户逐项决定；
4. P3 展示层身份按获批范围中性化；
5. 文件发现与目录残留分开统计；
6. 真正写入前经过范围审批和精确预览/token 审批；
7. 每个原始对象进入项目外事务备份，并留下恢复证据。

## 2. 实验对象与隐私边界

- 对象：一套生产导向的 Next.js AI SaaS Starter。
- 包管理器：pnpm。
- Git：目标目录本身不是 Git 仓库，因此恢复能力不能依赖 Git。
- 保留能力：认证、计费、积分、AI 生成、后台管理、国际化、法律页面与可用 Demo。
- 公开材料：只展示聚合数字和合成示意图；不公开完整 audit、diff、token、源码或购买项目身份。

## 3. 方法

完整流程为：

```text
discover → audit → Gate 1 → external preview → Gate 2 → apply → verify
```

- Audit 阶段只读扫描，识别 source identity、文件发现和目录发现。
- Gate 1 记录品牌模式、P1 迁移/回滚、P2 删除与重命名、行级语义编辑及空目录路径。
- Preview 在项目外生成完整副本和审批 artifacts，真实项目保持不变。
- Gate 2 把当前源码状态、Preview 状态、操作集合、目录 device/inode/state 和五份审批 artifact 哈希绑定到 64 位 token。
- Apply 只接受当前精确 token；原始对象通过 no-clobber 原子移动进入外部备份。
- Verify 重新扫描，不隐藏有意保留的发现；退出码 3 表示“仍有发现”，不是自动改写为成功。

目录数字没有并入 523 → 227。获批目录在 Preview 和 Apply 两个边界均被证明为空；它的普通父目录继续保留。

### 4.3 已批准的真实操作

- 72 个输出路径发生变化；
- 4 项明确删除；

## 6. 事务与恢复证据

空目录不是通过 `rmdir` 丢弃，而是通过不可覆盖的原子移动进入项目外 backup。恢复材料记录：

- 获批目录路径与执行顺序；
- 原目录 device/inode、mode 和内容状态；
- backup 位置；
- `restore.json` 恢复顺序；
- `reverse.diff` 人类可读逆向说明；
- `apply-result.json` 的实际修改、删除、重命名与目录清理列表。

无法安全回滚时，系统保留外部 backup 并报告 incomplete rollback，不覆盖并发出现的外来对象。

公开合成实验分别提交了错误令牌和 Preview 后已经过期的旧令牌，两次都在项目写入、backup 和 `apply-result.json` 产生前拒绝；inventory 保持一致，没有部分执行获批删除、重命名或目录清理。这两次拒绝不是在购买的真实项目里故意制造破坏。

v0.1.2 保留事务失败自动回滚、外部字节备份、`restore.json`、`reverse.diff` 和 `apply-result.json`，但没有一键恢复命令。发生无法安全覆盖外来对象的恢复冲突时，系统保留 backup 并报告 incomplete rollback。

低风险不等于零风险。用户明确批准的错误决策仍然可能造成不希望的修改，因此 Git 或可靠备份、两道人工检查和 Apply 后验证不能省略。

## 7. 为什么这个 Skill 不是一次性提示词

普通对话当然可以从一句“帮我去掉 Starter 痕迹”开始；本项目本身就是从这句话开始的。Skill 的价值不在于替用户多写一句提示词，而在于把后续容易遗漏的工程判断固化并复用：

- 相同的 P0–P3 风险模型；
- 缺品牌信息时暂停或使用固定中性占位符；
- 两道不可跳过的审批门；
- hash、状态、目录身份与 token 绑定；
- 外部 Preview、事务备份与恢复证据；
- 文件和目录双指标；
- 可重复测试、压力实验和公开隐私边界。

换一个用户、项目或对话，Agent 不需要重新“临场发明”这套流程。

## 8. 局限

- v0.1 自动 Apply 只支持具备 POSIX no-follow 能力的 macOS/Linux；Windows 暂不支持。
- 项目与外部 run directory 必须位于同一文件系统，才能保证原子移动语义。
- Skill 不替用户发明真实品牌、公司、域名或迁移计划。
- 它不会保证所有关键词消失；保留 P0/P1/P2/P3 是设计行为。
- 它不能替代法律审查、生产密钥管理或真实支付迁移演练。

## 9. 结论

真实验收达成：文件发现 523 → 227，目录残留 1 → 0，P0/LICENSE 保持，真实功能保留，没有新增测试失败，构建成功，并具备可验证的外部恢复材料。

更准确的结论不是“把 Starter 三个字删干净”，而是：把每一处痕迹变成可分类、可选择、可审批、可回滚、可解释的工程决策。
